# プロンプトテンプレート集

Claude CoworkとClaude in Chromeでブラウザ自動化を行う際の実践的なプロンプトテンプレート。

## 基本テンプレート

### 情報抽出

```
【目的】[対象サイト]から[情報の種類]を抽出する

【対象URL】
- [URL1]
- [URL2]

【抽出項目】
- [項目1]: [説明/セレクタのヒント]
- [項目2]: [説明/セレクタのヒント]
- [項目3]: [説明/セレクタのヒント]

【出力形式】
[CSV/JSON/Markdown表]形式で保存
ファイル名: [filename]

【注意事項】
- ページが完全に読み込まれてから抽出を開始
- データが見つからない場合は「N/A」と記録
- [X]件ごとに進捗報告
```

### フォーム自動入力

```
【目的】[サイト名]のフォームに[データソース]の情報を入力

【前提条件】
- 対象サイト: [URL]
- 入力データ: [ファイルパス]

【フィールドマッピング】
| フォームフィールド | データ列 |
|------------------|---------|
| [field1]         | [column1] |
| [field2]         | [column2] |
| [field3]         | [column3] |

【手順】
1. [URL]を開く
2. ログインが必要な場合は私に通知（手動でログイン）
3. 各レコードに対して:
   a. 「新規作成」ボタンをクリック
   b. 上記マッピングに従ってフィールドを入力
   c. 「保存」をクリック
   d. 成功を確認してから次へ

【エラー処理】
- エラー発生時は該当レコードをスキップしてログに記録
- 3回連続エラーで一時停止して報告

【完了報告】
処理件数、成功件数、エラー件数をサマリーで報告
```

### メール管理

```
【目的】受信トレイを整理する

【分類ルール】
1. 送信者が[ドメイン/リスト]に含まれる → [アクション]
2. 件名に「[キーワード]」を含む → [アクション]
3. [日数]日以上前の既読メール → [アクション]
4. 添付ファイル付きの未読メール → [アクション]

【アクション定義】
- archive: アーカイブに移動
- star: スターを付ける
- label:[名前]: ラベルを適用
- skip: 何もしない

【確認フロー】
1. まず条件に該当するメールをカウントして報告
2. 私の確認を待つ
3. 確認後、実際の処理を実行

【除外条件】
- 今日受信したメールは処理しない
- スター付きメールは変更しない
```

### 競合調査

```
【目的】競合[N]社の[調査項目]を比較分析

【対象企業・URL】
| 企業名 | URL |
|-------|-----|
| [A社] | [URL] |
| [B社] | [URL] |
| [C社] | [URL] |

【収集項目】
- [項目1]: [取得場所のヒント]
- [項目2]: [取得場所のヒント]
- [項目3]: [取得場所のヒント]

【出力】
1. 比較表（CSV形式）
2. 分析サマリー（Markdown形式）
   - 各社の特徴
   - 価格帯の傾向
   - 差別化ポイント

【注意】
- 価格は税抜き/税込みを明記
- 取得日時を記録
- 情報が見つからない項目は「情報なし」と記載
```

### ウェブアプリテスト

```
【目的】[機能名]の動作確認

【対象】
- URL: [localhost:ポート or URL]
- 機能: [テスト対象の機能]

【テストケース】
1. [ケース1]: 
   - 操作: [具体的な操作]
   - 期待結果: [期待する表示/動作]

2. [ケース2]:
   - 操作: [具体的な操作]
   - 期待結果: [期待する表示/動作]

3. [エラーケース]:
   - 操作: [不正な入力/操作]
   - 期待結果: [エラーメッセージ/バリデーション]

【確認項目】
- コンソールにエラーが出ていないか
- UIが正しく表示されているか
- レスポンス時間が許容範囲か

【報告形式】
| テストケース | 結果 | 備考 |
|-------------|------|------|
| [ケース1]   | PASS/FAIL | [詳細] |
```

## 安全確認プロンプト

破壊的操作の前に使用:

```
【確認要求】
以下の操作を実行する前に、対象と影響を報告してください:

操作内容: [削除/送信/公開/購入など]
対象: [具体的な対象]
影響範囲: [何が変わるか]

実行前に私の明示的な承認を待ってください。
「実行」と入力するまで待機すること。
```

## Chrome連携の指示追加

Cowork + Chrome連携時の追記テンプレート:

```
【ブラウザ操作が必要な部分】
Claude in Chromeを使用して以下を実行:
- [ブラウザでの操作1]
- [ブラウザでの操作2]

【ローカルファイル操作】
Coworkのファイルアクセスで以下を実行:
- [ファイル操作1]
- [ファイル操作2]

【連携ポイント】
[ブラウザから取得したデータ]を[ローカルファイル]に保存
```

## ショートカット登録用テンプレート

Chrome拡張の「/」コマンドで使用するショートカット:

```
/daily-report
毎日の業務報告を作成:
1. Gmailで今日送受信したメールを確認
2. カレンダーで本日の予定を確認
3. 完了したタスク、進行中のタスク、明日の予定をまとめる
4. daily-report-YYYY-MM-DD.mdとして保存
```

```
/inbox-zero
受信トレイをゼロに:
1. ニュースレター → アーカイブ
2. 自動通知 → 既読にしてアーカイブ
3. 要返信 → スター付けてラベル「要返信」
4. 残りは内容確認して適切に処理
処理前に件数報告、確認後に実行
```

```
/research-topic [トピック]
[トピック]について調査:
1. 関連サイト5つ以上を検索
2. 各サイトから要点を抽出
3. 調査結果をresearch-[topic]-[date].mdに保存
4. 参照URLリストを末尾に追加
```
